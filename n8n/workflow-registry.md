## Local Exports Present In Repo

All local workflow exports live under `n8n/workflows/`.

- Each workflow has a unique n8n ID (generated by n8n)
- Naming: `[Phase]-[Kurzname]-v[Version]` (e.g. `P1-telegram-intake-v2`)
- Minimum logging: `event_id` + `timestamp`
- Error handler: `P1-error-handler-v1` (ID `ezKkyglJhrTwPi8n`) should be assigned to all v2 workflows
- Contract/Schema: `n8n/contracts/telegram-intake-event.json`

## Testing-Specific Routing

- Webhook test ingress path: `test-intake`
- Test task lists: `NEXT_TEST`, `WAITING_TEST`
- Test vault path: `obsidian-vault/00_INBOX_TEST/`
- Safety mode: test traffic may write only to the test sinks above
- Telegram is smoke-only; mass runs use the dedicated webhook transport

## Notes

- The generator of record is `n8n/scripts/generate-v2-workflows.js`.
- The parser/normalizer source of truth is `n8n/api/command-parser.js`.
- Do not hand-edit only the exported JSON files. Regenerate them after changing parser or workflow structure.
